# Scimiette Sviluppatrici

Organization GitHub dove si builda, si rilascia e si risolve.
Niente fuffa, niente cap.

## Chi siamo

Siamo scimmiette sviluppatrici con mentalita semplice:
shipping costante, qualita alta, responsabilita vera.
Quando va bene prendiamo aura.
Quando qualcosa cade, restiamo locked in finche torna su.

## Team

| Ruolo in org | Cosa copre |
| --- | --- |
| Caposcimmia del Merge | Direzione tecnica, architettura e scelte che reggono in produzione. |
| Domatore del Backlog | Priorita prodotto, scope chiaro e ticket pronti da sviluppare. |
| Sciamano delle API | Servizi backend, dati e integrazioni senza magie instabili. |
| Custode della Pipeline | CI CD, monitoring e incident response senza panico. |

## Regole no cap

1. Merge solo con test e contesto.
2. Ogni fix veloce deve avere piano di cleanup.
3. Se in locale va ma in produzione no, siamo cooked e si interviene subito.
4. Retro sempre: stesso errore due volte non si replica.

## Workflow

- Issue chiare: problema, impatto, come riprodurre.
- PR piccole e leggibili, review diretta e utile.
- Deploy con rollback pronto.
- Incidenti gestiti con ownership, non con scuse.

## Recruiting

Se ti piace costruire bene e prenderti responsabilita, ci siamo.
Se vuoi solo fare scena, skip.
